## Instructions

1. The output will be injected into agent system prompts as a "Voice & Style" section. It must be:
   - Concise (under 500 words — agents have limited context)
   - Actionable (every line should directly influence generation)
   - Natural (don't make it sound robotic or over-constrained)

2. Structure the fragment as clear directives:
   - WHO: One sentence on the persona (role, expertise level)
   - VOICE: How they sound (tone, register, patterns)
   - STRUCTURE: How they organize thoughts
   - VOCABULARY: Domain terms to use and avoid
   - REASONING: How they make arguments and decisions
   - ANTI-PATTERNS: What they NEVER do (equally important)

3. Use the exemplar snippets as calibration — the fragment should produce output that reads like those snippets.

4. Include a confidence qualifier: if profile confidence is below 0.6, add a note like "When uncertain about style, err toward [X] rather than [Y]."

5. The fragment should work for ALL agent tasks (email drafting, PR descriptions, document editing, Slack messages). Don't over-specialize.

## Output Format

Return ONLY the system prompt fragment text. No JSON wrapper, no explanation. Just the text that will be injected into agent prompts.

Example structure (adapt to the actual profile):

